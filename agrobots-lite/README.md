# 🌾 AgroBots Lite

Упрощённая версия симулятора автономного роботизированного агрохозяйства
(хакатон «AgroDataCode», РГАУ–МСХА, сюжет №3 — *роботизированные системы и
автономная техника: маршрутизация роботов и обслуживание*).

Это «облегчённая» сборка основного проекта AgroBots: сохранены **ООП-иерархия
роботов** и **алгоритмы** (A\*, приоритетная очередь нарядов, устойчивая
сортировка плана), но функционал намеренно сокращён до объёма, посильного за
один рабочий день. Оставлен **только графический интерфейс** (tkinter) и
подсистема **заряда / автозарядки** роботов.

## 🚀 Запуск

Нужен только **Python 3.10+** — сторонние библиотеки не требуются
(`tkinter` входит в стандартную библиотеку):

```bash
python main.py
```

> Для графического режима нужен рабочий дисплей. На этой машине запускайте
> Python-сборкой из Microsoft Store:
> ```bash
> "$LOCALAPPDATA/Microsoft/WindowsApps/python3.13.exe" main.py
> ```
> При проблемах с кириллицей в путях добавьте `PYTHONUTF8=1`.

Самопроверка ядра (без pytest):

```bash
python tests/test_core.py
```

## 🎮 Интерфейс

Окно делится на области: слева — **поле** (клетки, зарядные станции `C`,
препятствия, маршруты роботов пунктиром, маркеры нарядов и роботы кружками с
иконкой типа, цветом статуса и полоской заряда); справа — вкладки
**Роботы / Наряды**; сверху — панель управления и строка статистики, снизу —
журнал событий.

| Кнопка | Действие |
|:------:|----------|
| **▶ Тик** | один тик симуляции |
| **⏩ День** | прожить один день |
| **📅 N дней** | прожить заданное число дней |
| **⟳ Автопрогон** | анимация по тикам (скорость — ползунком) |
| **➕ Робот** | добавить робота (сеялка / комбайн / разведчик) |
| **📋 Наряд** | создать задачу (посев / сбор / осмотр / полив) |
| **↺ Сброс** | перезапустить симуляцию с начала |

Клик по клетке поля выделяет её — координаты подставляются в диалоги
добавления робота и наряда.

## 🧩 Архитектура

Слоистый пакет, зависимости направлены вниз (`ui → core → models`):

```
agrobots-lite/
├── main.py                     # запуск GUI
├── requirements.txt            # сторонних зависимостей нет
├── tests/test_core.py          # самопроверка алгоритмов и ООП
├── ДОКУМЕНТАЦИЯ.md             # техническая документация
└── agrolite/
    ├── models/                 # доменные сущности (без алгоритмов)
    │   ├── field.py            # Field (2D-сетка) + Cell + CellKind
    │   ├── task.py             # Task, TaskKind, TaskStatus
    │   └── robot.py            # Robot ◄─ Seeder / Harvester / Scout
    ├── core/                   # алгоритмическое ядро
    │   ├── pathfinding.py      # A* по сетке
    │   ├── algorithms.py       # merge_sort, nearest
    │   ├── scheduler.py        # Dispatcher (очередь приоритетов)
    │   └── simulation.py       # пошаговый движок (тик/день, заряд)
    ├── ui/
    │   └── gui.py              # графический интерфейс (tkinter)
    └── scenarios.py            # демо-мир и стартовые наряды
```

## 🔬 Что сохранено и что упрощено

**Сохранено (ядро):**
- ООП с наследованием и полиморфизмом: `Robot` → `Seeder` / `Harvester` /
  `Scout` (у каждого свои `HANDLES`, `move_cost`, `work_cost`).
- Алгоритмы: **A\*** для маршрутизации, **двоичная куча** (`heapq`) как
  очередь приоритетов диспетчера, **устойчивая сортировка слиянием**
  (`merge_sort`) для плана дня, поиск ближайшего исполнителя (`nearest`).
- Подсистема **заряда**: роботы тратят энергию на движение и работу и при
  низком заряде сами прокладывают маршрут (A\*) к ближайшей станции.
- Графический интерфейс на tkinter.

**Убрано ради простоты (есть в полной версии AgroBots):**
- случайные события (погода, поломки) и seed-детерминизм;
- склад запчастей и техобслуживание (износ, ремонт);
- консольный интерфейс на `rich`/`colorama`, режим `--demo`, меню команд;
- собственная структура `LinkedList` (журнал ТО).

Подробное описание модулей — в [ДОКУМЕНТАЦИЯ.md](ДОКУМЕНТАЦИЯ.md).
