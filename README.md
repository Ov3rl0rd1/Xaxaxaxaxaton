# 🤖 AgroBots — симулятор роботизированного агрохозяйства

Консольное приложение для внутривузовского хакатона **«AgroDataCode»**
(РГАУ–МСХА им. К.А. Тимирязева, 2026).

**Сюжет №3:** *Роботизированные системы и автономная техника
(маршрутизация роботов, обслуживание).*

Парк автономных роботов обслуживает поле: роботы получают наряды из общей
очереди приоритетов, прокладывают маршрут по сетке поля (алгоритм **A***),
выполняют агрооперации, самостоятельно ездят на зарядку и техобслуживание.
В симуляцию встроены случайные события: погода влияет на расход энергии,
техника изнашивается и может ломаться.

## 🚀 Запуск

Нужен **Python 3.10+** и две библиотеки — **rich** и **colorama**:

```bash
pip install -r requirements.txt
# или напрямую:
pip install rich colorama
```

```bash
# графический интерфейс (tkinter) — режим по умолчанию
python main.py

# старый консольный режим (меню команд + rich)
python main.py --console

# неинтерактивная демонстрация: прожить 3 дня и показать итог
python main.py --demo 3

# выбрать зерно генератора случайных событий
python main.py --demo 5 --seed 7
```

`tkinter` входит в стандартную библиотеку Python — отдельная установка не
нужна (для графического режима требуется лишь рабочий дисплей).

> **Важно (для данной машины):** интерпретатор по умолчанию
> `C:\Program Files (x86)\Python` — это встраиваемая сборка без `pip` и без
> `rich`/`colorama`. Запускайте проект Python-сборкой из Microsoft Store, где
> библиотеки уже установлены:
> ```bash
> "$LOCALAPPDATA\Microsoft\WindowsApps\python3.13.exe" main.py --demo 3
> ```
> На сборках в режиме `safe_path` запускайте именно `main.py` — он сам
> добавляет каталог проекта в `sys.path`. При проблемах с кириллицей в путях
> используйте `PYTHONUTF8=1`.

Самопроверка логики:

```bash
python tests/test_core.py     # без pytest
pytest -q                     # если установлен pytest
```

## 🎮 Графический интерфейс

Окно делится на области: слева — **поле** (клетки, маршруты роботов
пунктиром, маркеры нарядов, роботы кружками с иконкой типа, цветом статуса и
полоской заряда); справа — вкладки **Роботы / Наряды / Склад**; сверху —
панель управления, под ней статистика, снизу — журнал событий.

| Кнопка | Действие |
|:------:|----------|
| **▶ Тик** | один тик симуляции |
| **⏩ День** | прожить один день |
| **📅 N дней** | прожить заданное число дней |
| **⟳ Автопрогон** | анимация по тикам (скорость — ползунком) |
| **➕ Робот** | добавить робота (сеялка / комбайн / разведчик) |
| **📋 Наряд** | создать задачу (посев / сбор / осмотр / полив) |
| **📦 Склад** | пополнить запчасти / расходники |
| **↺ Сброс** | перезапустить симуляцию с начала |

Клик по клетке поля выделяет её — координаты подставляются в диалоги
добавления робота и наряда.

## 🧩 Архитектура (ООП)

```
agrobots/
├── models/                 # доменные сущности
│   ├── field.py            # Field (2D-сетка) + Cell + CellKind
│   ├── task.py             # Task, TaskKind, TaskStatus
│   ├── robot.py            # Robot ◄─ Seeder / Harvester / Scout (наследование)
│   └── warehouse.py        # ServiceDepot (склад запчастей)
├── structures/
│   └── linked_list.py      # собственный односвязный список (журнал ТО)
├── core/                   # алгоритмическое ядро
│   ├── pathfinding.py      # A* и BFS (маршрутизация)
│   ├── algorithms.py       # merge_sort, binary_search, nearest
│   ├── scheduler.py        # Dispatcher (очередь приоритетов)
│   ├── events.py           # генератор случайных событий
│   └── simulation.py       # пошаговый движок симуляции
├── ui/                     # интерфейсы
│   ├── gui.py              # графический интерфейс (tkinter) — по умолчанию
│   ├── theme.py            # тема rich, общий Console, init colorama
│   └── console.py          # карта, таблицы, панели, дашборд
├── scenarios.py            # демо-мир и стартовое наполнение
├── app.py                  # интерактивное меню
└── __main__.py             # точка входа (python -m agrobots)
```

> 📄 Подробное описание каждого модуля, класса и метода вынесено в отдельный
> файл **[ДОКУМЕНТАЦИЯ.md](ДОКУМЕНТАЦИЯ.md)** — исходный код намеренно не
> содержит комментариев и докстрингов.

**Связи между классами:**
- **Наследование:** `Seeder`, `Harvester`, `Scout` → `Robot`
  (полиморфизм: `can_handle`, `move_cost`, `work_cost`).
- **Композиция:** робот владеет маршрутом (`deque`) и журналом ТО
  (`LinkedList`); `Field` состоит из объектов `Cell`.
- **Агрегация:** `Simulation` агрегирует роботов, поле, склад и диспетчер;
  `Dispatcher` агрегирует задачи в очереди приоритетов.

## 🗂️ Структуры данных (и обоснование выбора)

| Структура | Где | Почему |
|-----------|-----|--------|
| `dict` (хэш-таблица) | склад запчастей `ServiceDepot.parts` | доступ/списание по ключу за **O(1)** |
| `deque` | маршрут робота `Robot.path` | извлечение следующего шага слева за **O(1)** |
| куча (`heapq`) | очередь задач `Dispatcher` | выбор самой приоритетной задачи за **O(log n)** |
| односвязный список | журнал обслуживания `Robot.log` | дешёвая вставка в хвост, последовательный обход истории |
| 2D-список | поле `Field.grid` | доступ к клетке по координатам за **O(1)** |

## ⚙️ Алгоритмы

1. **Маршрутизация (обход графа/сетки):** `A*` с эвристикой Манхэттена,
   запасной — `BFS`. Поле = граф с 4-связностью. *(pathfinding.py)*
2. **Сортировка:** устойчивая `merge sort` **O(n log n)** — построение
   «плана задач на день» по приоритету и сроку. *(algorithms.py)*
3. **Поиск:** `binary search` **O(log n)** по отсортированному каталогу
   запчастей + линейный поиск ближайшего робота/станции. *(algorithms.py)*
4. **Планирование задач:** жадное назначение через очередь приоритетов —
   самая срочная задача → ближайшему способному роботу. *(scheduler.py)*

## 🌦️ Бизнес-логика и события

- **Заряд:** движение и работа тратят батарею; при ≤25 % робот едет на станцию.
- **Износ:** растёт от работы; при ≥80 % робот едет на ТО (списывается деталь
  `gearbox`); вероятность поломки растёт с износом.
- **Погода:** дождь/жара увеличивают расход энергии.
- **Поломки:** случайны; сорванный наряд возвращается в очередь, к роботу
  выезжает мобильный сервис.
